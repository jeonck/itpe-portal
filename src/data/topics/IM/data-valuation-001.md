# 정의
데이터 가치 평가는 기업이 보유하거나 활용하는 데이터가 창출하는 경제적, 비즈니스적 가치를 측정하고 분석하는 프로세스입니다. 데이터가 단순한 정보 자원을 넘어 기업의 핵심 자산이자 미래 성장의 동력으로 인식되면서, 데이터의 가치를 객관적으로 평가하는 방법론의 중요성이 증대되고 있습니다.

## 특징
- 평가 필요성:
    - 데이터 기반의 투자 의사결정 지원.
    - 기업의 무형 자산으로서 데이터의 가치 인정 및 재무제표 반영.
    - 데이터 매매, 라이선싱 등 데이터 경제 활성화.
    - 데이터 거버넌스 및 관리 전략 수립에 기여.
- 주요 평가 방법론:
    - 원가 기반 접근법 (Cost-Based Approach):
        - 데이터를 수집, 저장, 처리, 관리하는 데 소요된 비용을 기준으로 가치를 평가합니다. (예: 데이터 취득 비용, 관리 비용, 품질 개선 비용)
        - 장점: 측정 용이.
        - 단점: 데이터의 실제 비즈니스 기여도 반영 어려움.
    - 시장 기반 접근법 (Market-Based Approach):
        - 유사한 데이터가 시장에서 거래되는 가격을 기준으로 가치를 평가합니다.
        - 장점: 시장 가치 반영.
        - 단점: 유사한 데이터의 시장 거래 사례가 부족할 경우 적용 어려움.
    - 수익 기반 접근법 (Income-Based Approach):
        - 데이터 활용을 통해 미래에 창출될 것으로 예상되는 수익을 현재 가치로 할인하여 평가합니다. (예: 데이터 기반 신규 서비스 매출 증가, 비용 절감 효과)
        - 장점: 데이터의 비즈니스 기여도 반영.
        - 단점: 미래 수익 예측의 불확실성, 할인율 설정의 주관성.
- 비재무적 가치: 데이터의 브랜드 가치 향상, 고객 만족도 증대, 의사결정 품질 개선, 혁신 가속화 등 정량화하기 어려운 무형의 가치도 고려해야 합니다.
- 데이터 품질과의 연계: 데이터의 가치는 품질(정확성, 완전성, 적시성, 일관성)에 크게 좌우됩니다.
