# 정의
RPA(Robotic Process Automation)는 사람이 반복적으로 수행하는 규칙 기반의 업무 프로세스를 소프트웨어 로봇(Bot)을 통해 자동화하는 기술입니다. 기존 시스템의 UI를 그대로 활용하여 인간의 행동을 모방함으로써, 시스템 통합 없이 빠르고 효율적으로 업무를 자동화하여 생산성과 효율성을 높입니다.

## 특징
- **업무 자동화**: 정형화되고 반복적인 업무(데이터 입력, 파일 이동, 이메일 처리, 보고서 생성 등)를 소프트웨어 로봇이 자동으로 수행.
- **비침습적 자동화**: 기존 IT 시스템의 API를 직접 연동하는 대신, 사용자 인터페이스(UI)를 통해 자동화를 구현하므로, 기존 시스템 변경 없이 적용 가능.
- **디지털 워커 (Digital Worker)**: RPA 로봇은 가상 직원처럼 24시간 365일 업무 수행이 가능하며, 오류 없이 정확하게 작업 처리.
- **주요 구성 요소**:
    - **RPA Designer**: 로봇의 작업 흐름을 설계하는 도구. (플로우차트, 드래그 앤 드롭 방식)
    - **RPA Robot (Runner)**: 설계된 작업을 실제 수행하는 로봇.
    - **RPA Controller (Orchestrator)**: 여러 로봇의 배포, 스케줄링, 모니터링 및 관리를 담당하는 중앙 통제 시스템.
- **장점**: 업무 처리 속도 향상, 휴먼 에러 감소, 비용 절감, 직원들의 고부가가치 업무 집중, 규제 준수 강화.
- **단점**: 비정형 업무 자동화 한계, 시스템 UI 변경 시 자동화 프로세스 수정 필요, 보안 취약점 발생 가능.
- **지능형 RPA (IPA)**: AI 기술(OCR, 자연어 처리, 머신러닝)을 RPA에 결합하여 비정형 데이터 처리 및 의사결정 기능까지 확장한 형태.
