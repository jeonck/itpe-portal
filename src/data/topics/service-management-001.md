# 정의
IT 서비스 관리(ITSM)는 고객에게 가치를 제공하기 위해 IT 서비스를 설계·제공·지원·개선하는 체계적 접근 방식으로, ITIL과 같은 모범 사례를 기반으로 합니다.

## 특징
- **ITSM(IT Service Management)**: IT를 기술 중심이 아닌 서비스 중심으로 관리하는 접근법으로, 비즈니스 가치 제공에 초점
- **ITIL v4**: IT 서비스 관리의 사실상 표준으로 4차원 모델(조직/사람, 정보/기술, 파트너/공급자, 가치흐름/프로세스)과 서비스 가치 시스템(SVS) 제시. 주요 프랙티스는 인시던트 관리, 문제 관리, 변경 관리, 서비스 데스크, 구성 관리 등
- **SLA/SLM**: SLA(Service Level Agreement)는 서비스 제공자와 고객 간 합의된 서비스 수준 협약, SLM(Service Level Management)은 SLA를 지속적으로 협상·합의·모니터링·보고·검토하는 프로세스. KPI 예시: 가용성 99.9%, 응답시간 5분 이내, 해결시간 4시간 이내
- **주요 프로세스**: 서비스 전략 → 서비스 설계 → 서비스 전환 → 서비스 운영 → 지속적 서비스 개선(CSI)
