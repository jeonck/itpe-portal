# 정의
공용 네트워크를 통해 암호화된 터널을 생성하여 안전한 사설 네트워크 연결을 제공하는 기술입니다.

## 동작원리
VPN은 다음과 같은 메커니즘으로 안전한 터널을 생성합니다:

**1. IPsec VPN 동작 (Site-to-Site VPN)**
네트워크 계층에서 IP 패킷을 암호화하여 사이트 간 안전한 연결을 제공합니다:

1단계: IKE (Internet Key Exchange) Phase 1
- 두 VPN 게이트웨이 간 관리 채널(ISAKMP SA) 수립
- 인증: 사전 공유 키(PSK) 또는 인증서 사용
- 암호화: AES, 3DES
- 해싱: SHA-256, MD5
- DH(Diffie-Hellman) 키 교환으로 세션 키 생성

2단계: IKE Phase 2
- 실제 데이터를 위한 IPsec SA 생성
- ESP(Encapsulating Security Payload) 또는 AH(Authentication Header) 선택

3단계: 데이터 전송
- ESP: IP 패킷 암호화 + 캡슐화
- 터널 모드: 전체 IP 패킷 암호화 (Site-to-Site)
- 전송 모드: 페이로드만 암호화 (Host-to-Host)

4단계: 복호화 및 전달
- 수신측 VPN 게이트웨이가 복호화 후 내부 네트워크로 전달

**2. SSL/TLS VPN 동작 (Remote Access VPN)**
웹 브라우저 기반으로 간편하게 접속합니다:

1) 클라이언트 → VPN 서버: HTTPS 연결 (포트 443)
2) TLS Handshake: 서버 인증서 검증, 세션 키 생성
3) 사용자 인증: ID/PW, 다중인증(MFA)
4) SSL 터널 수립: 애플리케이션 데이터를 SSL로 암호화
5) 웹 기반 접근: 브라우저에서 내부 리소스 접속

특징:
- 클라이언트 소프트웨어 불필요 (브라우저만으로 접속)
- 방화벽 통과 용이 (HTTPS 포트 443 사용)

**3. WireGuard 동작 (차세대 VPN)**
현대적인 암호화 기술로 빠르고 간단한 VPN을 제공합니다:

1) 공개키 기반 인증
   - 각 피어(peer)는 공개키/개인키 쌍 보유
   - 상대방의 공개키를 미리 등록

2) Noise Protocol Framework로 핸드셰이크
   - 세션 키 생성, 재생 공격 방지

3) ChaCha20-Poly1305로 패킷 암호화
   - UDP 기반 전송, 빠른 성능

4) Cryptokey Routing
   - IP 주소와 공개키를 매핑하여 라우팅

특징:
- 코드 4,000줄 (OpenVPN 70,000줄 대비 단순)
- 빠른 속도, 낮은 지연시간
- 리눅스 커널에 통합

**4. Split Tunneling**
VPN 트래픽과 일반 트래픽을 분리하여 성능을 최적화합니다:

Full Tunnel Mode
- 모든 트래픽이 VPN을 경유 (보안 최대)

Split Tunnel Mode
- 내부 리소스 접근만 VPN 사용
- 인터넷 트래픽은 직접 연결 (성능 향상)

## 특징
- 목적: 원격 접속, 데이터 암호화 및 기밀성, IP 주소 우회/익명화
- 프로토콜: IPsec (Site-to-Site), SSL/TLS VPN (Remote Access), PPTP (레거시), L2TP/IPsec, WireGuard
- IPsec: OSI 3계층(네트워크) 암호화, Site-to-Site VPN, IKE 키 교환
- SSL VPN: OSI 7계층(응용), 웹 브라우저 기반, 클라이언트 간편, HTTPS 포트 443
- WireGuard: 최신 프로토콜, 단순(4천 줄), 빠름, 강력한 암호화(ChaCha20)
- 터널링: 데이터를 암호화하여 캡슐화, 공용 네트워크에서 사설 네트워크처럼 동작
- Split Tunneling: VPN 트래픽과 일반 인터넷 트래픽 분리
- 사용 사례: 재택근무 원격 접속, 본사-지사 간 안전한 연결, 공용 WiFi 보안, 지역 차단 우회
