# 정의
여러 서버에 네트워크 트래픽을 분산하여 가용성과 성능을 향상시키는 기술로, L4와 L7 계층에서 동작합니다.

## 특징
- L4 Load Balancing: 전송 계층 (TCP/UDP), IP/포트 기반 분산
- L7 Load Balancing: 응용 계층 (HTTP/HTTPS), URL/헤더/쿠키 기반 분산
- 알고리즘: Round Robin, Least Connections, IP Hash, Weighted
- Health Check: 서버 상태 모니터링, 장애 서버 자동 제외
- Session Persistence (Sticky Session): 세션 유지
- AWS: ALB (L7), NLB (L4), CLB (Classic)
- 기타: NGINX, HAProxy, F5
- 사용 사례: 대규모 웹 서비스, MSA, Auto Scaling

## 최신 트렌드
- Service Mesh
- Global Load Balancing
