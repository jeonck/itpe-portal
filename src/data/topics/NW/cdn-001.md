# 정의
지리적으로 분산된 서버 네트워크를 통해 콘텐츠를 사용자와 가까운 위치에서 제공하여 속도와 가용성을 향상시키는 서비스입니다.

## 동작원리
CDN은 다음과 같은 메커니즘으로 콘텐츠를 빠르게 전달합니다:

1. 콘텐츠 캐싱 및 배포 프로세스

1단계: 사용자 요청
- 사용자가 www.example.com/image.jpg 요청
- DNS가 가장 가까운 CDN 엣지 서버 IP 반환

2단계: 캐시 확인 (Cache Hit/Miss)
- Cache Hit: 엣지 서버에 콘텐츠 존재
  → 즉시 사용자에게 전달 (Origin 접근 불필요)

- Cache Miss: 엣지 서버에 콘텐츠 없음
  → Origin Server에서 콘텐츠 가져오기 (Pull)
  → 엣지 서버에 캐싱
  → 사용자에게 전달

3단계: 캐시 저장
- TTL(Time to Live) 기반으로 캐시 유지
- Cache-Control 헤더에 따라 캐시 정책 적용
- LRU(Least Recently Used) 알고리즘으로 캐시 관리

2. 지리적 라우팅 (GeoDNS)
사용자와 가장 가까운 엣지 서버로 연결합니다:

1) 사용자가 DNS 쿼리 전송
2) GeoDNS가 사용자 IP의 지리적 위치 파악
3) 가장 가까운 PoP(Point of Presence)의 IP 반환
4) 낮은 지연시간으로 콘텐츠 제공

3. 캐싱 전략

캐시 정책 설정
- Cache-Control: max-age=3600 (1시간 캐싱)
- Cache-Control: no-cache (매번 검증)
- Cache-Control: private (브라우저만 캐싱)
- ETag: 콘텐츠 버전 식별

캐시 무효화 (Purge/Invalidation)
- 콘텐츠 업데이트 시 캐시 즉시 삭제
- 방법: URL 기반, Tag 기반, 전체 삭제

4. Origin Shield
Origin 서버 부하를 줄이기 위한 중간 캐시 계층입니다:

1) 여러 엣지 서버 → Origin Shield (중간 캐시)
2) Origin Shield → Origin Server (필요시만)
3) Origin 요청 수 90% 이상 감소

5. DDoS 방어
분산 아키텍처로 DDoS 공격을 완화합니다:

- Anycast 네트워크: 공격 트래픽을 전 세계 PoP에 분산
- Rate Limiting: 과도한 요청 차단
- WAF (Web Application Firewall): 악성 트래픽 필터링

6. 동영상 스트리밍 최적화
Adaptive Bitrate Streaming (ABR)
- 네트워크 속도에 따라 화질 자동 조절
- HLS (HTTP Live Streaming), DASH 프로토콜

Chunked Transfer
- 동영상을 작은 조각(Chunk)으로 분할
- 각 조각을 엣지에서 캐싱

## 특징
- 동작 원리: 엣지 서버에 콘텐츠 캐싱, 가장 가까운 서버에서 제공, GeoDNS 라우팅
- 장점: 지연 시간 감소(50~90%), 대역폭 절감, 가용성 향상, DDoS 방어, Origin 부하 감소
- 캐싱 전략: TTL (Time to Live), Cache-Control 헤더, ETag, LRU 알고리즘
- 주요 업체: Cloudflare (200+ 도시), AWS CloudFront, Akamai, Fastly
- PoP (Point of Presence): CDN 엣지 서버가 위치한 데이터센터
- Origin Server: 원본 콘텐츠를 저장하는 서버
- Purge/Invalidation: 캐시 무효화, URL/Tag 기반
- 사용 사례: 동영상 스트리밍(Netflix, YouTube), 정적 파일 배포, API 가속, 게임 패치
