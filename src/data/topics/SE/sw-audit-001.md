# 정의
정보시스템 개발 프로젝트의 품질 및 관리 적정성을 독립적으로 검증하는 소프트웨어 감리 제도로, 한국의 3단계 감리가 대표적 기술.

# 특징
- 감리 목적: 품질 보증, 위험 관리, 사업 관리 적정성 검증
- 3단계 감리: 계획/설계 단계, 개발 단계, 시험/종료 단계
- 감리 기준: 품질 기준, 사업관리 기준, 기술 기준
- 점검 항목: 요구사항, 설계, 구현, 테스트, 형상관리, 일정/비용 관리
- 감리 프로세스: 계획 → 실행 → 보고 → 개선조치
- 감리원 자격: 정보관리기술사, 전자계산기조직응용기술사 등
- 법적 근거: 소프트웨어진흥법, 전자정부법
- 감리 보고서: 지적사항, 개선권고사항, 종합의견

## 절차
소프트웨어 감리는 다음과 같은 3단계 프로세스로 수행됩니다:

1단계: 계획/설계 단계 감리
프로젝트 초기에 요구사항과 설계의 적정성을 검증:

1. 감리 계획 수립
   - 감리 범위, 일정, 투입 인력 계획
   - 감리 기준: 소프트웨어 감리 기준(정보통신산업진흥원)
   - 점검 항목 선정: 프로젝트 규모/특성에 따라 맞춤형 점검표 작성

2. 요구사항 점검
   - 요구사항 명세서(SRS) 완전성 검토: 누락된 기능 요구사항 확인
   - 요구사항 추적성 매트릭스: 사용자 요구 → 시스템 요구사항 추적
   - 비기능 요구사항: 성능, 보안, 가용성 목표 명확성
   - 요구사항 변경 관리: 변경 이력 및 영향 분석 프로세스 확인

3. 시스템 설계 점검
   - 아키텍처 설계: 시스템 구조, 계층 분리, 인터페이스 정의 적정성
   - 데이터베이스 설계: ERD, 정규화, 인덱스 설계 검토
   - 인터페이스 설계: 외부 시스템 연계 방식, 데이터 포맷 명세
   - 보안 설계: 인증/인가, 암호화, 접근 통제 방안
   - 설계 산출물 품질: UML 모델, 상세 설계서 완성도

4. 사업 관리 점검
   - 일정 관리: WBS, 마일스톤, 일정 준수율
   - 비용 관리: 예산 대비 집행 현황, Cost Overrun 리스크
   - 인력 관리: 투입 인력 적정성, 핵심 인력 확보 여부
   - 위험 관리: 리스크 식별, 대응 계획

5. 지적사항 도출 및 보고
   - 중대 지적: 프로젝트 성공에 심각한 영향 (예: 핵심 요구사항 누락)
   - 권고사항: 개선 필요 항목 (예: 설계 문서 보완)
   - 개선 조치 기한 설정: 다음 감리 시점까지 시정 요구

2단계: 개발 단계 감리
구현 중간 단계에서 개발 프로세스와 품질을 점검:

1. 이전 감리 지적사항 이행 점검
   - 1단계 지적사항 조치 완료 여부 확인
   - 미이행 시 재지적 및 강도 높은 조치 요구

2. 코드 품질 점검
   - 코딩 표준 준수: 네이밍 컨벤션, 주석, 코드 리뷰 수행 여부
   - 정적 분석 도구: SonarQube 등으로 코드 품질 측정
   - 보안 취약점: SQL Injection, XSS 등 OWASP Top 10 점검
   - 코드 복잡도: Cyclomatic Complexity, 중복 코드 비율

3. 단위/통합 테스트 점검
   - 테스트 계획: 테스트 전략, 범위, 일정
   - 테스트 케이스: 요구사항 커버리지, 경계값 테스트
   - 테스트 자동화: JUnit, Selenium 등 자동화 비율
   - 결함 관리: 버그 추적, 재테스트, 회귀 테스트

4. 형상 관리 점검
   - 형상 관리 도구: Git, SVN 사용 적정성
   - 버전 관리 전략: 브랜치 전략, 태깅 규칙
   - 변경 통제: 소스 코드 변경 승인 프로세스

5. 프로젝트 진행 상황
   - 일정 지연: 지연 원인 분석, 만회 계획
   - 산출물 완성도: 개발 완료율 대비 문서화 수준

3단계: 시험/종료 단계 감리
프로젝트 완료 전 최종 품질과 운영 준비 상태를 점검:

1. 시스템 테스트 점검
   - 통합 테스트: 전체 시스템 통합 후 기능/비기능 요구사항 검증
   - 성능 테스트: 부하 테스트, 스트레스 테스트 결과 분석
   - 보안 테스트: 침투 테스트, 취약점 스캔 결과
   - UAT (사용자 인수 테스트): 실사용자 테스트 결과 및 승인

2. 운영 준비 점검
   - 배포 계획: 배포 전략, 롤백 계획, 데이터 마이그레이션
   - 운영 매뉴얼: 시스템 운영 가이드, 장애 대응 절차
   - 사용자 교육: 교육 계획, 교육 자료 완성도
   - 유지보수 인수인계: 소스 코드, 문서, 개발 환경 이관

3. 산출물 완전성 점검
   - 필수 산출물: 요구사항 명세서, 설계서, 소스 코드, 테스트 결과서, 사용자 매뉴얼
   - 산출물 품질: 최신성, 일관성, 완전성

4. 프로젝트 종료 평가
   - 목표 달성도: 요구사항 구현율, 일정/예산 준수
   - 품질 평가: 결함 밀도, 테스트 커버리지
   - 교훈 정리: 성공 요인, 개선 사항, 향후 프로젝트 적용 방안

5. 최종 감리 보고서 작성
   - 종합 의견: 프로젝트 성공 가능성 판단
   - 조건부 통과: 특정 항목 개선 후 운영 전환 허가
   - 불합격: 중대 결함으로 재감리 요구
