# 정의
정보시스템 개발 프로젝트의 품질 및 관리 적정성을 독립적으로 검증하는 소프트웨어 감리 제도로, 한국의 3단계 감리가 대표적입니다.

## 특징
- 감리 목적: 품질 보증, 위험 관리, 사업 관리 적정성 검증
- 3단계 감리: 계획/설계 단계, 개발 단계, 시험/종료 단계
- 감리 기준: 품질 기준, 사업관리 기준, 기술 기준
- 점검 항목: 요구사항, 설계, 구현, 테스트, 형상관리, 일정/비용 관리
- 감리 프로세스: 계획 → 실행 → 보고 → 개선조치
- 감리원 자격: 정보관리기술사, 전자계산기조직응용기술사 등
- 법적 근거: 소프트웨어진흥법, 전자정부법
- 감리 보고서: 지적사항, 개선권고사항, 종합의견
