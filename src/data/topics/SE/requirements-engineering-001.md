# 정의
소프트웨어 개발의 성공을 위해 이해관계자의 요구사항을 체계적으로 추출, 분석, 명세, 검증, 관리하는 프로세스 기술.

## 특징
- 요구사항 추출(Elicitation): 인터뷰, 설문, 브레인스토밍, 프로토타이핑, 관찰
- 요구사항 분석(Analysis): 타당성, 일관성, 완전성, 모호성 제거
- 요구사항 명세(Specification): SRS(Software Requirements Specification) 작성
- 요구사항 검증(Validation): 리뷰, 인스펙션, 워크스루, 프로토타이핑
- 요구사항 관리: 변경 관리, 추적성(Traceability), 버전 관리
- 기능 요구사항: 시스템이 제공해야 할 기능
- 비기능 요구사항: 성능, 보안, 사용성, 신뢰성
- Use Case, User Story, Persona, Story Mapping

## 절차
요구공학 프로세스는 다음과 같은 단계로 수행됩니다:

1. 요구사항 추출 (Elicitation)
   이해관계자의 니즈를 파악하는 단계로, 다양한 기법을 활용합니다:
   - 인터뷰 및 설문조사로 직접적인 요구사항 수집
   - 워크샵 및 브레인스토밍을 통한 협업적 요구사항 도출
   - 프로토타이핑으로 시각적 피드백 수집
   - 관찰 및 작업 분석을 통한 암묵적 요구사항 발견

2. 요구사항 분석 (Analysis)
   수집된 요구사항을 정제하고 구조화하는 단계:
   - 타당성 분석: 기술적, 경제적, 법적 실현 가능성 검토
   - 우선순위 결정: MoSCoW (Must, Should, Could, Won't) 기법 적용
   - 모호성 제거: 명확하고 검증 가능한 문장으로 재작성
   - 충돌 해결: 상충되는 요구사항 간 조정 및 협상

3. 요구사항 명세 (Specification)
   분석된 요구사항을 공식 문서로 작성:
   - SRS (Software Requirements Specification) 문서 작성
   - Use Case Diagram, Sequence Diagram 등 UML 모델링
   - User Story 형식으로 Agile 백로그 작성
   - 기능 요구사항과 비기능 요구사항 구분하여 명세

4. 요구사항 검증 (Validation)
   명세된 요구사항이 이해관계자의 니즈를 올바르게 반영했는지 확인:
   - 리뷰 및 인스펙션: 동료 검토, 이해관계자 승인
   - 워크스루: 단계별 요구사항 시나리오 검토
   - 프로토타이핑: 실제 구현 전 목업으로 검증
   - 테스트 케이스 작성: 요구사항이 테스트 가능한지 확인

5. 요구사항 관리 (Management)
   프로젝트 전 생애주기 동안 요구사항을 추적하고 관리:
   - 변경 관리: 요구사항 변경 요청 접수, 영향 분석, 승인 프로세스
   - 추적성 매트릭스: 요구사항 → 설계 → 구현 → 테스트 연결
   - 버전 관리: 요구사항 문서의 변경 이력 관리
   - 도구 활용: JIRA, Confluence, Azure DevOps 등

이 과정은 반복적(Iterative)이며, Agile 환경에서는 스프린트마다 지속적으로 수행됩니다.
