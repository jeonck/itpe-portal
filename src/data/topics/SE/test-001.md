# 정의
소프트웨어의 품질을 검증하고 결함을 발견하기 위한 체계적인 활동으로, 다양한 기법과 전략을 사용합니다.

## 동작원리
소프트웨어 테스팅은 V-모델을 따라 개발 단계별로 대응하는 테스트를 수행합니다:

1. 단위 테스트 (Unit Test)
   개별 함수나 메서드의 동작을 검증하는 가장 작은 단위의 테스트:
   - 화이트박스 기법: 코드 내부 구조를 알고 모든 경로 테스트
   - 구문 커버리지: 모든 코드 라인이 최소 한 번 실행되도록 테스트
   - 분기 커버리지: 모든 조건문의 참/거짓 경로 테스트
   - 경로 커버리지: 가능한 모든 실행 경로 테스트
   - 도구: JUnit, PyTest, Jest

2. 통합 테스트 (Integration Test)
   여러 모듈이 함께 동작할 때의 인터페이스와 상호작용을 검증:
   - 빅뱅 통합: 모든 모듈을 한 번에 통합 후 테스트
   - 상향식 통합: 하위 모듈부터 점진적으로 통합 (Driver 필요)
   - 하향식 통합: 상위 모듈부터 점진적으로 통합 (Stub 필요)
   - API 테스트: REST API 엔드포인트 호출 및 응답 검증

3. 시스템 테스트 (System Test)
   전체 시스템이 요구사항을 만족하는지 검증:
   - 기능 테스트: 블랙박스 기법으로 기능 요구사항 검증
   - 비기능 테스트: 성능, 보안, 사용성, 호환성 검증
   - 부하 테스트: 동시 사용자 증가 시 성능 측정
   - 스트레스 테스트: 시스템 한계점 파악

4. 인수 테스트 (Acceptance Test)
   사용자 관점에서 시스템이 업무 요구사항을 충족하는지 검증:
   - UAT (User Acceptance Test): 실제 사용자가 직접 테스트
   - 알파 테스트: 개발사 내부 사용자 테스트
   - 베타 테스트: 외부 일부 사용자 대상 테스트

블랙박스 테스팅 기법:
- 동등 분할: 입력 값을 유사한 그룹으로 나누어 대표 값 테스트
- 경계값 분석: 입력 범위의 경계(최소, 최대, 경계±1) 집중 테스트
- 상태 전이: 시스템 상태 변화와 전이 조건 검증

## 특징
- 블랙박스 테스팅: 동등 분할, 경계값 분석, 상태 전이
- 화이트박스 테스팅: 구문 커버리지, 분기 커버리지, 경로 커버리지
- V-모델: 각 개발 단계에 대응하는 테스트 단계 정의
- 정적 분석: 코드 리뷰, 정적 코드 분석 도구
- 동적 분석: 실행 기반 테스트 (단위, 통합, 시스템, 인수)
