# 정의
소프트웨어의 품질을 검증하고 결함을 발견하기 위한 체계적인 활동으로, 다양한 기법과 전략을 사용합니다.

## 특징
- 블랙박스 테스팅: 동등 분할, 경계값 분석, 상태 전이
- 화이트박스 테스팅: 구문 커버리지, 분기 커버리지, 경로 커버리지
- V-모델: 각 개발 단계에 대응하는 테스트 단계 정의
- 정적 분석: 코드 리뷰, 정적 코드 분석 도구
- 동적 분석: 실행 기반 테스트 (단위, 통합, 시스템, 인수)
