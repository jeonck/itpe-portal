# 정의
정보시스템 감리는 정보시스템의 효율성, 신뢰성, 안전성을 확보하기 위해 제3자 전문가가 개발·운영 전반을 점검·평가하는 제도로, 소프트웨어 진흥법에 근거합니다.

## 특징
- 3단계 감리: 1) 착수단계 감리 - 사업계획, 요구사항 타당성 점검 2) 중간단계 감리 - 설계·구현 적정성, 진척도 점검 3) 종료단계 감리 - 시험·인수, 산출물 완성도 점검
- 감리 프레임워크: 감리계획 수립 → 감리 수행(인터뷰, 문서검토, 시스템 점검) → 감리결과 보고 → 개선권고 → 이행확인
- 감리기준: 1) 사업관리 분야 - 사업계획, 일정/예산 관리 2) 요구사항 관리 분야 - 요구사항 정의, 추적성 3) 설계 분야 - 아키텍처, 상세설계 4) 구현/시험 분야 - 코딩표준, 단위/통합시험 5) 전환/보안 분야 - 데이터 전환, 보안성
- 감리 유형: 정기감리(법정감리), 수시감리(요청감리), 특정감리(취약점 집중), 사후감리(준공 후)
