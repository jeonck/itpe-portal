# 정의
코드 리뷰(Code Review)는 소프트웨어 개발 과정에서 작성된 코드를 다른 개발자가 검토하여 잠재적인 오류, 버그, 취약점, 비효율적인 코드 등을 발견하고 개선하는 활동입니다. 동료 검토(Peer Review)는 코드 리뷰를 포함하여 설계 문서, 요구사항 명세서 등 프로젝트의 다양한 산출물을 동료들이 상호 검토하여 품질을 향상시키는 포괄적인 활동을 의미합니다.

## 특징
- 목표:
    - 결함 발견 및 수정: 소프트웨어 개발 초기 단계에서 결함을 발견하고 수정하여 개발 비용 절감.
    - 품질 향상: 코드의 가독성, 유지보수성, 효율성, 보안성을 높여 소프트웨어 품질 향상.
    - 지식 공유: 팀원 간의 코드 및 설계에 대한 이해 증진, 기술 공유 및 역량 강화.
    - 표준 준수: 코딩 표준, 설계 원칙 등의 준수 여부 확인.
- 코드 리뷰 (Code Review):
    - 주체: 주로 개발자 간에 이루어집니다.
    - 대상: 소스 코드.
    - 방법:
        - 오버더숄더 (Over-the-shoulder): 두 명의 개발자가 한 화면을 보며 검토. (짝 프로그래밍 포함)
        - 이메일 기반: 변경된 코드를 이메일로 보내 검토.
        - 도구 기반: Git 기반의 코드 리뷰 툴(GitHub Pull Request, GitLab Merge Request, Crucible 등)을 사용하여 변경 사항을 검토하고 피드백 제공.
        - 모임 기반: 코드 작성자가 코드를 설명하고 여러 명의 동료가 검토.
    - 장점: 초기 단계 결함 발견, 코드 품질 향상, 지식 전파, 팀워크 강화.
- 동료 검토 (Peer Review):
    - 범위: 코드뿐만 아니라 설계 문서, 요구사항 명세서, 테스트 계획서 등 프로젝트의 모든 산출물.
    - 종류:
        - 워크스루 (Walkthrough): 작성자가 주도하여 동료들에게 산출물을 설명하고 피드백을 받음. 비공식적인 성격.
        - 인스펙션 (Inspection): 체계적인 절차와 역할을 가지고 결함을 찾는 공식적인 검토. (사전 준비, 회의, 사후 조치) 가장 엄격한 형태.
- 효과:
    - 개발 프로세스 초기에 결함을 발견하여 수정 비용 절감.
    - 개발 표준 및 베스트 프랙티스 준수율 향상.
    - 개발 팀의 역량 강화 및 학습 문화 조성.
    - 소프트웨어 유지보수 용이성 증대.
