# 정의
갈등 관리 기법은 프로젝트 팀 내외부에서 발생하는 다양한 갈등 상황을 효과적으로 해결하고, 긍정적인 방향으로 전환하여 프로젝트 목표 달성에 기여하도록 하는 방법론입니다. 프로젝트 환경에서 갈등은 필연적으로 발생하며, 이를 어떻게 관리하느냐에 따라 프로젝트 성과에 큰 영향을 미칩니다.

## 특징
- **갈등의 원인**:
    - **자원 부족**: 제한된 자원(예산, 인력, 장비)을 둘러싼 경쟁.
    - **우선순위 충돌**: 작업 또는 목표에 대한 우선순위가 다를 때.
    - **개인의 가치관 및 성격 차이**: 팀원 간의 개인적인 견해나 성격 차이.
    - **의사소통 부족**: 정보의 오해나 왜곡, 불충분한 정보 공유.
    - **역할 및 책임 불명확**: 누가 무엇을 해야 하는지 명확하지 않을 때.
    - **절차 및 정책의 차이**: 작업 방식이나 절차에 대한 이견.
- **갈등 관리 전략 (토마스-킬만 갈등 모드)**: 개인의 관심(Assertiveness)과 관계의 관심(Cooperativeness) 두 축을 기준으로 5가지 유형으로 분류됩니다.
    1.  **회피 (Avoid / Withdrawal)**:
        - **특징**: 갈등을 외면하거나 미루는 전략. 자신과 상대방의 관심 모두 낮음.
        - **장점**: 사소한 갈등, 시간 벌기.
        - **단점**: 문제 해결되지 않고 재발 가능.
    2.  **수용 / 양보 (Accommodate / Smooth)**:
        - **특징**: 자신의 관심보다 상대방의 관심을 우선하여 양보하는 전략.
        - **장점**: 관계 유지, 갈등 완화.
        - **단점**: 자신의 목표 달성 미흡.
    3.  **강요 / 지시 (Force / Compete)**:
        - **특징**: 자신의 관심을 강하게 주장하고 상대방의 관심은 무시하는 전략. (Win-Lose)
        - **장점**: 신속한 결정, 위기 상황.
        - **단점**: 관계 악화, 반발 초래.
    4.  **타협 (Compromise)**:
        - **특징**: 자신과 상대방이 모두 어느 정도 양보하여 중간 지점에서 합의하는 전략.
        - **장점**: 공정한 해결, 신속한 합의.
        - **단점**: 최적의 해결책이 아닐 수 있음, 모두에게 100% 만족 주지 못함.
    5.  **협력 / 문제 해결 (Collaborate / Confront)**:
        - **특징**: 자신과 상대방의 관심 모두를 충족시키기 위해 함께 문제를 해결하는 전략. (Win-Win)
        - **장점**: 최적의 해결책 도출, 관계 강화.
        - **단점**: 시간과 노력이 많이 소요.
- **프로젝트 관리자의 역할**: 갈등의 원인을 파악하고, 상황과 이해관계자의 특성을 고려하여 가장 적절한 갈등 관리 기법을 선택하고 적용하는 것이 중요합니다.
