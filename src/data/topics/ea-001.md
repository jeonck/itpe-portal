# 정의
EA(Enterprise Architecture)는 조직의 업무, 데이터, 응용, 기술 등 전사 자원을 통합적으로 관리하는 체계로, 비즈니스 목표와 IT 자원을 정렬하여 최적화하는 프레임워크입니다.

## 특징
- **현행/목표 아키텍처**: As-Is(현행) 아키텍처 분석 → Gap 분석 → To-Be(목표) 아키텍처 설계 → 이행 로드맵 수립의 단계적 접근
- **4대 참조 모델**: 1) BA(Business Architecture) - 업무 프로세스, 조직 구조 2) DA(Data Architecture) - 데이터 표준, ERD, 데이터흐름 3) AA(Application Architecture) - 응용 시스템 구성, 인터페이스 4) TA(Technology Architecture) - 인프라, 플랫폼, 네트워크
- **EA 프레임워크**: TOGAF(ADM 방법론), Zachman Framework(6x6 매트릭스), FEAF(연방정부), Gartner EA
- **핵심 활동**: 아키텍처 거버넌스, 표준화, 재사용성 확보, 중복 투자 방지, 상호운용성 확보
