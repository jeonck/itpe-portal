# 정의
HPC(High Performance Computing, 고성능 컴퓨팅)는 매우 복잡한 계산 문제나 대규모 데이터 처리가 요구되는 과학, 공학, 비즈니스 분야에서 최고 수준의 컴퓨팅 성능을 제공하는 시스템을 의미합니다. 수많은 프로세서가 병렬로 연결되어 초고속 연산을 수행하며, 슈퍼컴퓨터가 HPC 시스템의 대표적인 예입니다.

## 특징
- **병렬 처리 (Parallel Processing)**:
    - **SIMD (Single Instruction Multiple Data)**: 하나의 명령어로 여러 데이터를 동시에 처리 (GPU, 벡터 프로세서).
    - **MIMD (Multiple Instruction Multiple Data)**: 여러 프로세서가 각각 다른 명령어로 다른 데이터를 동시에 처리 (클러스터, 멀티코어 CPU).
- **분산 컴퓨팅 (Distributed Computing)**:
    - 여러 대의 컴퓨터가 네트워크로 연결되어 하나의 작업을 분담하여 처리.
    - **클러스터 컴퓨팅**: 고속 네트워크로 연결된 여러 대의 일반 컴퓨터(노드)를 단일 시스템처럼 활용. (MPI, OpenMP)
    - **그리드 컴퓨팅**: 지리적으로 분산된 컴퓨팅 자원을 공유하여 거대한 가상 슈퍼컴퓨터처럼 활용.
- **아키텍처**:
    - **대규모 병렬 처리 (MPP, Massively Parallel Processing)**: 수천 개의 프로세서가 독립적인 메모리를 가지고 고속 인터커넥트 네트워크로 연결.
    - **공유 메모리 시스템 (SMP, Symmetric Multiprocessing)**: 여러 CPU가 하나의 공유 메모리에 접근. (UMA, NUMA)
- **성능 지표**:
    - **FLOPS (Floating Point Operations Per Second)**: 초당 부동 소수점 연산 횟수. (PFLOPS, EFLOPS)
    - **처리량 (Throughput)**: 단위 시간당 처리할 수 있는 작업량.
    - **지연 시간 (Latency)**: 데이터 처리 요청 후 응답까지 걸리는 시간.
- **활용 분야**:
    - **과학 및 공학**: 기후 모델링, 신약 개발, 핵융합 시뮬레이션, 유체 역학 분석 (CFD).
    - **AI**: 대규모 딥러닝 모델 학습.
    - **금융**: 고빈도 거래(HFT), 리스크 모델링.
    - **제조**: 제품 설계 및 시뮬레이션 (CAE).
- **엑사스케일 컴퓨팅 (Exascale Computing)**: 1초에 10^18번의 연산(ExaFLOPS)이 가능한 컴퓨팅 시스템을 목표. (예: Frontier, Aurora)
- **클라우드 HPC**: 클라우드 환경에서 HPC 자원을 온디맨드(on-demand)로 활용. (비용 효율성, 유연성).
