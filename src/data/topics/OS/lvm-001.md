# 정의
물리적 디스크를 논리적 볼륨으로 추상화하여 유연한 디스크 관리를 가능하게 하는 Linux 스토리지 관리 기술.

## 특징
- 구조: Physical Volume (PV) → Volume Group (VG) → Logical Volume (LV)
- PV (물리 볼륨): 실제 디스크 또는 파티션 (/dev/sda1 등)
- VG (볼륨 그룹): 여러 PV를 묶은 스토리지 풀
- LV (논리 볼륨): VG에서 할당된 가상 파티션, 파일 시스템 생성
- 장점: 동적 크기 조정, 디스크 추가 용이, 스냅샷, 여러 디스크 통합
- Snapshot: 특정 시점의 LV 상태 저장, 백업 및 복구
- 명령어: pvcreate, vgcreate, lvcreate, lvextend, lvreduce
- 사용 사례: 서버 스토리지 관리, 데이터베이스 볼륨, 백업
